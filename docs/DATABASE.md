# Database Schema & Entity Framework Strategy

## 1. Engine & ORM
- **Database Engine:** PostgreSQL
- **ORM:** Entity Framework Core (Code-First Migrations)
- **State:** Migrations are stored in `Data/Migrations` and are applied sequentially.

## 2. Design Philosophy
- **Soft Deletes:** (Where applicable) preferring `IsPublished` boolean toggles over destructive row drops to maintain historical data integrity.
- **Strict Typing:** Utilizing Data Annotations and Fluent API in `OnModelCreating` to enforce strict `MaxLength` constraints, preventing database bloat and ensuring UI uniformity.
- **SEO Primacy:** Required fields like `Slug` on specific entities guarantee that public frontend routing and technical SEO JSON-LD injection have reliable data.

## 3. Entity Specifications

### A. `Faq` (Frequently Asked Questions)
Used to populate the public FAQ accordion page and dynamically inject `FAQPage` JSON-LD schema for search engine rich snippets.

| Property | Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `Id` | `int` | Primary Key, Auto-Increment | Unique identifier. |
| `Question` | `string` | Required, MaxLength 255 | The public-facing question. |
| `Answer` | `string` | Required | Stores sanitized HTML generated by the locked-down Quill.js editor. |
| `DisplayOrder` | `int` | Default: 0 | Controls the sorting sequence on the public UI. |
| `IsPublished` | `bool` | Default: false | Draft vs. Live visibility toggle. |
| `CreatedAt` | `DateTime` | Required | Timestamp of record creation (UTC). |
| `UpdatedAt` | `DateTime?` | Nullable | Timestamp of last modification (UTC). |

### B. `TechTip` (IT Blog / Guides)
Used to populate the Tech Tips section and dynamically inject `Article` or `HowTo` JSON-LD schema.

| Property | Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `Id` | `int` | Primary Key, Auto-Increment | Unique identifier. |
| `Title` | `string` | Required, MaxLength 255 | The article headline. |
| `Slug` | `string` | Required, MaxLength 255, Unique | URL-friendly hyphenated string for SEO routing (e.g., `how-to-reset-router`). |
| `Content` | `string` | Required | Stores sanitized HTML generated by Quill.js. |
| `VideoUrl` | `string?` | Nullable | Optional embed link for YouTube/Vimeo supplements. |
| `Category` | `string?` | Nullable, MaxLength 100 | Categorization tag (e.g., "Cybersecurity", "Networking"). |
| `IsPublished` | `bool` | Default: false | Draft vs. Live visibility toggle. |
| `CreatedAt` | `DateTime` | Required | Timestamp of record creation (UTC). |
| `UpdatedAt` | `DateTime?` | Nullable | Timestamp of last modification (UTC). |

## 4. Seeding Strategy
During early UI development and isolated testing, mock data is injected via `AppDbContext.OnModelCreating` using the `HasData()` method. This ensures that frontend tables and modals can be visually verified before the physical PostgreSQL connection string is finalized.

*Note: Seed data IDs are hardcoded (e.g., `Id = 1`) to prevent sequence conflicts during Entity Framework migrations.*
